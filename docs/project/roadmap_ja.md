# ロードマップ

このドキュメントはlawkitの将来の開発計画を概説します。

## バージョン 2.1 (2025年Q1)

### パフォーマンス強化
- [ ] **ストリーミング分析**: メモリより大きなファイルの真のストリーミング
- [ ] **GPU高速化**: 大規模計算のCUDA/OpenCLサポート
- [ ] **並列処理**: 全統計法則の強化されたマルチスレッド
- [ ] **メモリ最適化**: 大規模データセットのメモリフットプリント削減

### 新しい統計法則
- [ ] **カイ二乗法則**: 適合度検定
- [ ] **べき法則**: スケールフリーネットワーク分析
- [ ] **ワイブル分布**: 信頼性と生存分析
- [ ] **ガンマ分布**: 連続確率分布

### 拡張フォーマットサポート
- [ ] **Apache Parquet**: 列指向ストレージ形式
- [ ] **Apache Arrow**: インメモリ列指向形式
- [ ] **HDF5**: 科学データ形式
- [ ] **データベースコネクタ**: 直接SQL データベース分析

## コミュニティ貢献

これらの目標達成のためのコミュニティ貢献を歓迎します：

### 貢献方法
- **機能リクエスト**: 新機能のためのissue提出
- **コード貢献**: 実装のプルリクエスト
- **ドキュメント**: ガイドと例の改善支援
- **テスト**: 新機能のベータテスト
- **フィードバック**: ユーザー体験とパフォーマンスフィードバック

### 優先分野
1. **新しい統計法則**: 追加法則の実装
2. **パフォーマンス最適化**: 速度とメモリ改善
3. **フォーマットサポート**: 新しいファイル形式パーサー
4. **ドキュメント**: 使用例とチュートリアル
5. **統合**: 人気ツール用コネクタ

## 技術スタック進化

### 現在のスタック
- **コア**: パフォーマンスと安全性のためのRust
- **CLI**: コマンドラインインターフェースのためのclap
- **パース**: データシリアライゼーションのためのserde
- **数学**: 統計計算のためのstatrs

### 将来の追加
- **GPU**: GPU高速化のためのcudf、rapids
- **Web**: WebAssemblyのためのwasm-bindgen
- **可視化**: チャートのためのplotly.js、d3.js
- **ML**: 機械学習のためのcandle、tch
- **クラウド**: 非同期ネットワーキングのためのtokio、hyper

## フィードバック

このロードマップに対するフィードバックを重視します：

- **GitHub Issues**: 機能リクエストと提案
- **ディスカッション**: 優先順位に関するコミュニティ入力
- **調査**: 定期的なユーザーニーズ評価
- **直接連絡**: メンテナーとのコミュニケーション

このロードマップは、ユーザーフィードバック、技術的制約、市場ニーズに基づいて変更される可能性があります。lawkitを価値あるものにするシンプルさとパフォーマンスを維持しながら、最も包括的な統計分析プラットフォームを構築することにコミットしています。
